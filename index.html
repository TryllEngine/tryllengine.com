Everything you need to know about the Tryll platform

8GB of VRAM is enough to run both a model and a game. However, more VRAM is better - graphics-intensive games at high settings may consume most of the memory, which can slow down the model. For the best experience, we recommend 12GB+ VRAM. Tryll automatically selects the most efficient model and quantization for the user's system to ensure smooth gameplay.

According to the Steam Hardware Survey (see VRAM section), about 2/3 of players already have 8GB+ VRAM cards, which are good enough to run local AI:

  • ~1/3 of players have 8GB VRAM
  • ~1/3 of players have 12GB+ VRAM

As open-source models get smaller and faster, and gamers continue upgrading hardware, this percentage grows - unlocking scalable deployment without cloud reliance.

Local AI advantages:

  • Zero cost per player - no API or infrastructure bills
  • Offline capable - your game keeps working without an internet connection
  • Private by design - no personal data leaves the device

Cloud-based AI adds legal and UX risks:

  • Immersion-breaking outages during play sessions
  • Legal compliance challenges (GDPR, AI Act, export laws)

Tryll's local-first approach is not just cheaper - it's safer, faster, and scalable to millions of players.

Tryll is designed so that AI is always additive to your game. AI inference runs separately from core gameplay, so if anything goes wrong, the game falls back to default behavior. We're building toward full process isolation, but even today the architecture ensures AI issues don't take down your game.

We're building Unreal and Unity plugins first, with an SDK API to follow. The platform is engine-agnostic at its core — the engine-specific plugins are convenience layers on top of the same underlying runtime. If you're working with a custom engine, reach out and we'll work with you on integration.

The key differences are architecture and scope:

  • On-device vs. cloud: Inworld and NVIDIA ACE rely on cloud infrastructure. Tryll runs entirely on the player's GPU — zero servers, zero per-user costs, no network round-trips.
  • Infrastructure vs. conversation: Most competitors focus on NPC dialogue. Tryll is a full AI runtime — LLM text, speech-to-text, text-to-speech, vision, and agent actions, all through one plugin.
  • No vendor lock-in: Tryll uses open-source models. You're not dependent on a proprietary API that can change pricing or shut down.

Think of it as the difference between renting a cloud GPU and having DirectX built into every player's machine.

Yes. Tryll supports any open-source model that runs in GGUF format. You can:

  • Use our curated model library (Llama, Mistral, Phi, Qwen, etc.)
  • Fine-tune an open-source model on your game's data and deploy it through Tryll
  • Swap models without changing any game code — the API stays the same

Tryll's Model Manager handles quantization, hardware detection, and runtime optimization automatically, regardless of which model you choose.

The AI assistant consumes VRAM only when generating responses. When you're not actively using it, the model is offloaded from VRAM and doesn't affect your gameplay at all. This means you get the full performance of your GPU for gaming, with AI assistance available on demand.

You can still run Tryll with a smaller model (like 3B parameters), but the AI quality won't be as good. We find models of 7B+ parameters to be reliable and fun to play with, while 3B models provide a noticeably less engaging experience. We recommend RTX 4070 or equivalent for the best results.

Tryll runs entirely on your device. We don't collect any personal data and have no access to your chat logs. Everything stays local. The only way for us to access your data is when you explicitly send it as part of a bug report or feedback.
